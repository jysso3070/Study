인터넷통신 api는 socket api를 이용한다

UDP - 순서가 없고 중간에 분실우려가 있다

TCP - 순서가 있다 (연결지향형)


주로 사용하는 메시지형식
  텍스트형식의 메시지 : 사람이 읽기 쉽다 디버깅에 용이. 그러나 보내야하는 데이터의 길이가 많아서 비효율적, 
  메시지를 받고서 strcmp를 통해 메시지를 번역해야해서 시간이 오래걸림

  바이너리 형식의 메시지 : 데이터 길이가 짧아서 효율적, 디버깅할때 힘들고 에러가 발생할때 사람이 쉽게 알아차리기
  어렵다 (mmorpg에서는 주로 사용 : 대역폭낭비가 적고, 변환이 불필요)
  
  메타데이터가 있는 형식 : 가독성이 좋다, 확장성이 높다, 데이터양이 커져서 효율이 떨어짐, 해석해야할 단어들도 많아짐
  바이너리 형식으로도 메타데이터를 포함시킬 수 있다. (모바일앱에서는 메타데이터가 유리 웹서버시스템을 그대로사용)

네트워크 주소 변환 Network Address Translation, NAT
  ipv4에서 아이피주소의 개수가 한계가 있어서 NAT라우터를 통해 여러대의 기기가 하나의 ip주소를 공유하여 사용한다
  라우터는 포트번호를 통해 패킷을 구분하여 전송
